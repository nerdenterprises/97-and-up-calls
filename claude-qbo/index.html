03
The Keys
Get Your Client ID & Client Secret
▾

Inside your new app are two values that unlock direct access to your QBO data. This is the moment. Copy them somewhere safe — but treat them like passwords.

1
Open your app from the Developer Dashboard
2
Click the Keys & credentials tab on the left
3
Make sure you're on the Development tab (not Production — that comes in Step 8)
4
Toggle Show credentials to reveal your keys
5
Copy your Client ID — long alphanumeric string — and paste it somewhere temporarily (Notepad, TextEdit)
6
Copy your Client Secret — do the same
📋
Also add your Redirect URI now. Still on the Keys & credentials page, find the Redirect URIs section and add: http://localhost:3000/callback

Type it exactly as written — nothing to personalise. localhost just means "this computer, right here." It's not a website you need to own or register. Your MCP server will start a tiny temporary web server on your machine during the login handshake, QBO sends you back to that address, and then it disappears. The address must match exactly between here and your .env file or QBO will reject the connection.
⚠
Never paste these into any chat window. Not Claude.ai. Not email. Not a shared doc. Store them in your .env file only — Step 4 walks you through that.
04
Security
Store Your Keys Safely in a .env File
▾

A .env file is just a plain text file that holds your keys safely on your machine. Claude Code creates it for you automatically when it builds your MCP server in Step 6 — you just need to know how to open it and paste your values in.

Here's what the file will look like when Claude Code creates it:

.env — what it looks like
QBO_CLIENT_ID=paste_your_client_id_here QBO_CLIENT_SECRET=paste_your_client_secret_here QBO_REDIRECT_URI=http://localhost:3000/callback QBO_ENVIRONMENT=sandbox

How to open and edit it after Claude Code creates it:

1
Navigate to your project folder in Finder (Mac) or File Explorer (Windows) — this is the folder Claude Code set up with you in Step 6
2
Show hidden files — the .env file starts with a dot which hides it by default
Mac: Press Cmd + Shift + .
Windows: Click View → Show → Hidden items
3
Right-click (Windows) or Cmd+Click (Mac) on the .env file → Open With → Notepad (Windows) or TextEdit (Mac)
4
Replace paste_your_client_id_here with your actual Client ID
5
Replace paste_your_client_secret_here with your actual Client Secret
6
Save and close the file
⚠
Never share this file. Don't email it. Don't upload it to Google Drive. Don't paste it into any chat — including Claude. The keys inside give full access to your QBO account.
ℹ
You don't need to create this file yourself. Claude Code does it in Step 6. Come back here when you're ready to paste your keys in.
05
One-Time Setup
Install Node.js
▾

Your MCP server is built in JavaScript and needs Node.js to run. Think of Node.js the same way you think of needing Excel installed before you can open a spreadsheet — it's just the runtime environment. You install it once, it runs in the background, and you never have to think about it again.

Installation is as simple as any other app:

1
Go to nodejs.org — you'll see a big download button front and center
2
Click the LTS version (the one that says "Recommended for most users") — it auto-detects whether you're on Mac or Windows
3
Open the downloaded file — it's a standard installer package (.pkg on Mac, .msi on Windows)
4
Click through: Continue → Continue → Agree → Install — accept all the defaults, don't change anything
5
Enter your computer password if prompted (Mac) — this is normal, it's just installing software
6
Click Close when it finishes — Node.js is now installed

Verify it worked — the one time you'll use Terminal:

This is the only time in all 12 steps you'll need to open a command window. You're typing one thing and looking for one response. That's it — and Seth will show you exactly what it looks like on screen.

Mac
Press Cmd + Space to open Spotlight → type Terminal → press Enter. A black window opens. Type node --version and press Enter.
Win
Press the Windows key, type cmd, press Enter. A black window opens. Type node --version and press Enter.
✓
What success looks like: You'll see a version number — something like v22.14.0. Any number is fine. That means Node.js is installed and working. You can close the window and never open it again.
📸 Screenshot — Seth's screen
Seth will show his Terminal / Command Prompt with the node --version command and the version number response so you know exactly what "it worked" looks like.
⚠
If you see "command not found" or "not recognized": Node.js didn't install correctly. Try restarting your computer first — that fixes it most of the time. If it still doesn't work, tell Claude Code: "I installed Node.js but node --version says command not found" and he'll walk you through the fix.

The Build
Have Claude Code Build Your MCP Server
▾

This is where it gets real. You're not writing a single line of code. You open Claude Code — the Code tab inside the Claude Desktop app — paste one prompt, and watch Claude build your entire MCP server. Every file. Every connection. Every tool. Done.

ℹ
What is an MCP server? It's a small program that runs quietly on your computer and acts as a bridge between Claude and QBO. Once it's running, Claude can talk to your QBO account directly — no exporting, no copy-pasting, no switching windows. You just chat in Claude Desktop and Claude talks to QBO in the background.

Open Claude Desktop → click the Code tab → paste this prompt:

The MCP Server Build Prompt — paste into Claude Code
I want to build a custom MCP server that connects Claude Desktop directly to my QuickBooks Online account. My Intuit Developer credentials are stored in a .env file — I have my Client ID, Client Secret, Redirect URI (http://localhost:3000/callback), and Environment (sandbox for now). Please build me a complete MCP server in Node.js that: 1. Handles QBO OAuth 2.0 authentication — opens a browser window for me to log in on first run, saves tokens locally so I don't have to log in every time, and automatically refreshes tokens before they expire 2. Exposes the following as MCP tools that Claude Desktop can call directly: - get_profit_and_loss (date range parameters) - get_balance_sheet (date range parameters) - get_cash_flow (date range parameters) - get_aged_receivables - get_aged_payables - get_invoices (filter by status, customer, date range) - get_overdue_invoices - query (accepts SQL-style QBO queries) - get_company_info 3. Creates a .env file with placeholders for my credentials that I can fill in myself 4. Creates a claude_desktop_config.json snippet showing exactly what to add to my Claude Desktop config file to connect this server 5. Includes a README with step-by-step setup instructions Create all files in a new folder called qbo-mcp-server in my home directory. Use the @modelcontextprotocol/sdk package. Include full error handling and clear console messages so I know what's happening when I run it.
✦
What Claude Code produces: A complete folder called qbo-mcp-server in your home directory containing all the code files, your .env template, and a config snippet for Claude Desktop. Everything you need — written and ready.

After Claude Code finishes building:

1
Go to your home folder and find the new qbo-mcp-server folder Claude Code just created
2
Open the .env file using the method from Step 4 (show hidden files → right-click → open with Notepad or TextEdit)
3
Paste in your Client ID and Client Secret from Step 3
4
Save and close the .env file
⚠
Claude Code vs. Claude.ai — important. Claude Code runs locally on your machine. When you reference your keys there, they stay on your machine — written directly into your .env file and going nowhere else. That's safe. Pasting your keys into a regular Claude.ai chat window in a browser is different — don't do that. Keys go into your .env file only, via Claude Code.
Step 6 Extension — Add Write Tools to Your Server
NEW

The build prompt above gives you a working server with read tools — pulling reports, querying data, reading invoices. That's powerful. But write tools — creating customers, sending invoices, logging bills — are a second step. If you already ran the build prompt, your server is running but write access isn't wired in yet. This is what we're adding today.

Before you run the write tools prompt, confirm your auth is alive:

1
Open a new chat in Claude Desktop
2
Type: "Pull my QBO company info" — if Claude returns your company name and details, your connection is live and your token is good. Move to the write tools prompt below.
3
If it returns an error or nothing, go to Claude Code and type: "My QBO MCP server connection isn't responding. Check the token file and refresh it if needed." Then try again.

Once your connection is confirmed — open Claude Code and paste this:

Write Tools Follow-Up Prompt — paste into Claude Code
I have an existing QBO MCP server in my qbo-mcp-server folder. It currently has read tools only. I need to add write tools so Claude can operate in my books, not just read from them. Please add the following new tools to the existing server: 1. create_customer — creates a new QBO customer Parameters: display_name (required), email (optional), phone (optional), billing_address (optional) Uses QBO v3 API: POST /v3/company/{realmId}/customer 2. update_customer — updates an existing customer's info Parameters: customer_id (required), display_name (optional), email (optional), phone (optional), billing_address (optional) Fetches the current customer first to get the SyncToken (required for updates), then sends a full update object Uses QBO v3 API: POST /v3/company/{realmId}/customer (with Id and SyncToken in the body) 3. create_product_service — creates a new product or service item Parameters: name (required), type (required — "Service" or "Inventory" or "NonInventory"), unit_price (optional), income_account_name (optional — maps to a QBO income account), description (optional) Uses QBO v3 API: POST /v3/company/{realmId}/item 4. create_invoice — creates and optionally sends an invoice Parameters: customer_ref (required), line_items (required — array of description, amount, and optional service date), due_date (optional), send_email (optional boolean) Uses QBO v3 API: POST /v3/company/{realmId}/invoice 5. create_bill — logs a vendor bill Parameters: vendor_name (required), amount (required), category (required — maps to QBO account name), transaction_date (required), memo (optional) Uses QBO v3 API: POST /v3/company/{realmId}/bill 6. create_account — creates a new account in the Chart of Accounts Parameters: name (required), account_type (required — e.g. "Income", "Expense", "Accounts Receivable", "Accounts Payable", "Bank", "Credit Card", "Other Current Asset", "Fixed Asset", "Other Current Liability", "Long Term Liability", "Equity"), account_sub_type (optional — QBO sub-type for the account), description (optional) Uses QBO v3 API: POST /v3/company/{realmId}/account 7. send_invoice_email — sends an existing invoice by email Parameters: invoice_id (required), email (optional — uses customer email on file if not provided) Uses QBO v3 API: POST /v3/company/{realmId}/invoice/{id}/send Important: - The existing OAuth tokens and auth logic must stay unchanged - The existing read tools must keep working - Add error handling that returns clear messages if a write fails - The scope com.intuit.quickbooks.accounting already covers write access — no OAuth changes needed After adding the tools, tell me how to restart the MCP server in Claude Desktop so the new tools are picked up.
✦
What happens after you run this: Claude Code edits your existing index.js (or equivalent) and adds the seven write tool handlers. Your read tools keep working. Restart Claude Desktop, open a new chat, and all the prompts in Step 9's write access section will be available.
⚠
Test in sandbox first. Make sure you're on QBO_ENVIRONMENT=sandbox in your .env before testing write tools. Creating a real customer or sending a real invoice to a real client by accident is not the demo you want to run on a Thursday morning.

Accessing a client's books:

A

Start Talking to Your Data — Just Chat
▾

Open the Chat tab in Claude Desktop. That's it. No Terminal. No commands. No exporting. You just type — and Claude talks to your QBO account in the background. Here are the prompts to start with.

✦
How this works: You type in Claude Desktop chat. Claude calls your MCP server tools behind the scenes. The server talks to QBO. Claude gets the data and responds. You see the analysis. The whole pipeline runs in seconds — invisible to you.
Year-over-Year Analysis
Pull my P&L for 2025 and 2024 and compare them. Where did I grow and where did I shrink? Where do I shine and where do I suck? Give me the honest CFO-level analysis — not just the numbers, tell me what they mean.
Quarterly Breakdown
Pull my P&L for each quarter of 2025 and give me a quarterly breakdown. Which quarter was my best? Which was my worst? Am I accelerating or decelerating? Then build me a visualization that shows all four quarters side by side.
Same-Quarter Comparison
Pull my P&L for Q1 2026 and Q1 2025 and compare them line by line. Where am I ahead of last year? Where am I behind? What's the single most important thing I should pay attention to — and what would you tell me to do differently in Q2?
Past Due Invoices — Who Owes Me?
Pull all my overdue invoices. Summarize them — total amount outstanding, how many clients, which ones are most overdue. Group them by 30, 60, and 90+ days past due. Who do I call first?
Past Due Invoices — Draft the Collections Emails
Based on those overdue invoices, draft a collections email for each client. Reference their specific invoice number and amount. Be polite for clients 30-59 days out, firmer for 60-89 days, and very direct for 90+ days. I'll review and send these myself.
Client Profitability
Pull my sales data for 2025 and tell me who my top 10 clients are by revenue. Rank them and show me the totals. Flag any client that represents more than 20% of my total revenue — that's a concentration risk I need to know about.
