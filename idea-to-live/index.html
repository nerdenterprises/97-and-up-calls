WHY NOT WIX OR SQUARESPACE?

To put your app at calc.yourpractice.com, you need to add a CNAME record to your domain's DNS. Wix, Squarespace, GoDaddy Website Builder, Weebly — these platforms own your DNS. They won't let you point a subdomain anywhere outside their walls. The fix is simple: keep your website where it is, but transfer your domain to a real registrar (GoDaddy registrar, Namecheap, Cloudflare). One-time setup. You'll thank yourself for the rest of your career.

STAGE 1 · PART 1 · APRIL 23

Build the App

Thursday's call. We build a real loan calculator from scratch in Claude Code.

One project folder, one prompt, one working tool. Then we iterate until it looks the way you want it to look. The loan calculator is a starting point — the workflow is what matters.

1

Open a project folder in Claude Desktop

Launch Claude Desktop. Click the Code tab at the top. Click Open folder… at the bottom. A file browser appears — navigate to where you want this project to live (Desktop, Documents, wherever), create a new folder named something simple like loan-calculator, select it, click Open.

Claude Code is now working inside that folder. You'll see a chat input at the bottom of the window that reads "Describe a task or ask a question." Every prompt goes there.

2

Build it with one prompt

Type this into the chat input and hit Enter. Claude Code reads what you asked for and writes the entire app in one shot.

PROMPTBuild me a single-page HTML loan calculator. Let the user enter loan amount, interest rate, term in years, and start date. Show monthly payment, total interest, total paid, and a full amortization schedule. Style it cleanly — soft shadows, rounded corners, professional palette. Mobile-friendly. Single index.html, no frameworks.
3

View it

Your new index.html file is right there in the project folder you opened — Claude Code shows it in the file tree on the left side of the window. Click it to see the code. Use the preview pane to see the calculator running right inside Claude Desktop.

Want to see it the way your clients will? Open your file browser (Finder on Mac, Explorer on Windows), navigate to the same folder, double-click index.html. Same calculator, real browser.

Either way: enter a loan amount, a rate, a term. Confirm the math. A real, interactive financial tool that didn't exist 60 seconds ago.

4

Iterate until it's yours

Push back. Add a comparison column. Add a chart. Match your firm's brand colors. Drag your logo into the chat alongside a prompt and Claude Code will place it in the header. Each prompt → file edits → see the change. This is the rhythm.

5

Make it yours — the closer

Spend the last few minutes pushing on details: layout, wording on each label, your firm's name in the header, a download button. Treat it like a real client tool — because by Tuesday, that's exactly what it'll be.

STAGE 2 · PART 2 · APRIL 28

Ship It Live

From your laptop to the public internet — on your own subdomain, with HTTPS.

You built the app in Claude Code. Now we put it on the internet at your own URL. Two equally valid paths to choose from — pick the one that fits your setup.

WHICH PATH IS RIGHT FOR YOU?

Path A — GitHub Pages. Best if you're using Claude Code with GitHub already connected, or if you want one consolidated workflow. Free forever, no usage limits, your code and your hosting in the same place.

Path B — Netlify. Best if you can't or don't want to connect GitHub to Claude Code, or if you want the simplest possible drop-and-deploy experience. Free tier with custom domains and HTTPS, friendlier UI, slight credit cap on the free plan that most members will never hit.

Either path gets you to the same place — a tool live at 97.yourpractice.com/your-tool/ with HTTPS, served free, ready to share. The skills (folder, CNAME, HTTPS) transfer perfectly between them.

PATH A
GitHub Pages
Code and hosting in one place. Free forever, no credit limits.
1

Create a new repository on GitHub

Log into GitHub → click the + icon top right → New repository. Name it something like 97-and-up-calls — one umbrella repo holds every visualization you'll ever ship.

Critical: set visibility to Public. GitHub Pages on free accounts only works with public repos, and the rendered URL is public anyway — private doesn't buy you anything. Skip README, .gitignore, and license. Click Create.

2

Prep the folder on your computer

Find the folder Claude Code built your tool in (the one you created in Part 1 — probably on your Desktop). Two small checks:

Folder name: if it isn't already your URL slug, rename it. loan-calculator becomes yoursite.com/loan-calculator/. Use lowercase-with-hyphens, no spaces, no capitals.

Main HTML file: open the folder and confirm there's a file called index.html. If the HTML file has a different name, rename it. GitHub Pages auto-serves index.html at folder URLs, giving you the clean URL instead of an ugly filename on the end.

Everything else Claude Code put in there: leave it alone. It all goes up together.

3

Drag-and-drop the folder into GitHub

On your empty repo page, click the "uploading an existing file" link in the blue Quick Setup box (or Add file → Upload files once the repo has content). Drag the whole folder from your computer into the drop zone — not just the HTML file inside it. GitHub preserves the folder structure automatically. Commit changes.

Why the folder, not just the file: for simple prompts like the loan calculator, Claude Code produces one index.html file and nothing else. But for more complex tools, it might produce separate CSS, JavaScript, or image files. The folder is the unit that works together — always drag the whole folder and you never have to think about which files matter.

4

Turn on GitHub Pages + set your custom subdomain

Repo → Settings → Pages. Source: Deploy from a branch. Branch: main / root. Save.

Scroll to Custom domain and enter the subdomain you want — e.g., 97.nerdenterprises.com. Click Save. It'll show a red X (DNS not pointed yet) — expected. Next step fixes it.

5

Add a CNAME at your registrar, wait, enforce HTTPS

Log into GoDaddy / Namecheap / Cloudflare. DNS management. Add a CNAME record:

CNAME RECORDType: CNAME Name: 97 (just the subdomain prefix) Value: username.github.io (no https://, no trailing slash) TTL: default

Propagation is usually 5–15 minutes. Back in GitHub Pages, click Check again until the green checkmark appears. Then check Enforce HTTPS. Let's Encrypt provisions a free SSL cert automatically. Visit your URL. Live.

TO UPDATE LATER (PATH A)

When you want to update a tool, navigate into its folder on GitHub (e.g., click into the loan-calculator folder). Click Add file → Upload files, and drag whatever files changed. GitHub replaces files that match names by filename — no need to delete first. For single-file tools, that's just a new index.html. For multi-file tools, drag all the changed files at once (or drag the whole folder again — any files that match names get replaced). New version live within a minute.

Part 3 shows the power-user version where Claude Code pushes changes directly and the site updates in 30 seconds without any drag-and-drop at all.

PATH B
Netlify
No GitHub required. Drop, connect, ship.
1

Add Netlify as a connector in Claude

Open Claude Desktop → Settings → Connectors → find Netlify in the list (or search if needed). Click Connect, authorize Claude through your Netlify account in the browser, return to Claude. This is the same kind of OAuth handshake we did for QBO — one-time setup, then Claude can talk to your Netlify account anytime.

Don't have a Netlify account yet? Go to app.netlify.com/signup first and sign up with GitHub, Google, or email. Free, takes a minute.

2

Prep the folder on your computer

Same step as Path A. Find your Claude Code project folder. Make sure the main HTML file is named index.html — Netlify, like GitHub Pages, auto-serves index.html as the entry point. Folder name doesn't matter for Netlify (you'll set the URL through Netlify's UI), but use a sensible lowercase-with-hyphens name anyway for your own sanity.

3

Drop the folder at Netlify Drop

Go to app.netlify.com/drop. You'll see a single big drop zone. Drag your project folder — the whole folder, not just the HTML inside — into the zone. Netlify uploads it, builds the site, and gives you a live URL like dazzling-pasteur-a1b2c3.netlify.app in about 10 seconds. Click it to confirm your tool loads.

Rename the site: in the dashboard for the new site, find Site configuration → Change site name. Pick something memorable like nerdenterprises-tools. URL becomes nerdenterprises-tools.netlify.app.

4

Add your custom subdomain in Netlify

In the site dashboard → Domain management → Add a domain you already own. Enter 97.yourpractice.com (or whatever subdomain you want). Click Verify. Netlify checks ownership and gives you DNS instructions on the next screen.

5

Add a CNAME at your registrar, wait, HTTPS auto-provisions

Log into GoDaddy / Namecheap / Cloudflare. DNS management. Add a CNAME record:

CNAME RECORDType: CNAME Name: 97 (just the subdomain prefix) Value: nerdenterprises-tools.netlify.app (your renamed Netlify URL, no https://, no trailing slash) TTL: default

Propagation is usually 5–15 minutes (often faster on Netlify than on GitHub). Back in Netlify, the domain status flips to Verified once DNS resolves. HTTPS auto-provisions via Let's Encrypt within minutes after that. Visit your URL. Live.

TO UPDATE LATER (PATH B)

Two ways. Manual update: go back to app.netlify.com/drop while logged in, drag the updated folder, and Netlify treats it as a new deployment of the same site (use the dashboard to deploy to your existing site rather than creating a new one). Power-user update (taught in Part 3): with the Netlify connector active in Claude, you can prompt Claude Code directly: "Deploy this folder to my Netlify site." It handles the upload and the URL updates in about 30 seconds.

Heads up on the free tier: Netlify Free includes 300 credits per month, with each deploy costing 15 credits. That's about 20 deploys per month before you hit the cap. For one-time tools and occasional updates, you'll never notice. For tools you're iterating on heavily, GitHub Pages (Path A) has no such limit.

WHAT TO BUILD NEXT

Seven Tools Worth Shipping

The loan calculator was the practice rep. Here's where it gets interesting.

These are tools that show off what you specifically know — and turn your website into a 24/7 demonstration of your expertise. Pick one. Build it next week. Drop the URL in the chat when you do.

USE CASE 01
Entity Comparison Calculator
Sole prop vs. LLC vs. S-corp vs. C-corp. Projected revenue and expenses in, side-by-side tax implications out. The conversation every prospect needs before deciding how to structure, and most accountants have with pen and paper.
Build me a single-page HTML entity comparison calculator for U.S. small businesses. Let the user enter projected annual revenue and expenses. Calculate and display side-by-side 2026 tax implications for four entity types: sole proprietorship (Schedule C with SE tax), single-member LLC (same as sole prop by default), S-corporation (assume a 50/50 salary/distribution split with FICA on salary only), and C-corporation (21% flat corporate tax plus qualified dividend tax at 15% on distributions). Show total tax burden and net take-home for each. Include a state income tax field defaulting to California at 9.3% and use it in all calculations. Use a clean side-by-side comparison table. Highlight the entity with the lowest total tax burden as "recommended." Include a clear disclaimer that this is for illustrative purposes only and actual tax depends on many factors. Style it cleanly with soft shadows, rounded corners, a professional palette suitable for an accounting practice. Mobile-friendly. Single index.html, no frameworks.
USE CASE 02
Reasonable Compensation Estimator
Officer role, industry, and location in, a defensible salary range out with plain-English methodology. The S-corp reasonable comp question has one right answer for each client and most owners get it wrong because they're guessing. A tool that produces a documented range helps you answer it once, defensibly, every time.
Build me a single-page HTML Reasonable Compensation Estimator for S-corporation officers. User selects their role from a dropdown (CEO, CFO, Operations Manager, Sales Manager, Consultant, Professional Services - Attorney/CPA/etc., Skilled Trade, Other), enters their industry, state, metropolitan area or ZIP code, years of experience, and approximate weekly hours. The tool returns a reasonable compensation range with a low, median, and high estimate, along with a plain-English explanation of the methodology — referencing the three IRS-approved approaches (cost approach, market approach, income approach) and noting which factors drove the range. Include a brief section on documentation best practices the owner should keep. Add a disclaimer that this is for planning purposes only and that actual reasonable compensation depends on specific facts and circumstances; recommend formal documentation through a qualified professional for audit defense. Professional palette, clean table layout, mobile-friendly. Single index.html, no frameworks.
USE CASE 03
Bookkeeping Cleanup Estimator
Six to eight questions about a prospect's current state, a ballpark quote range out. Replaces the "just give me a quote" email chain with a qualifier a prospect runs through on their own before reaching you.
Build me a single-page HTML questionnaire that estimates the scope and cost of a bookkeeping cleanup project. Ask 6-8 questions with appropriate input types: how many bank accounts and credit cards combined (number), how many months behind on reconciliation (number), whether payroll is involved (yes/no), whether sales tax is involved (yes/no), whether inventory is involved (yes/no), approximate monthly transaction volume (dropdown: 0-50, 51-200, 201-500, 501-1000, 1000+), current entity type (dropdown), and whether prior-year tax returns are filed and matching QBO (yes/no/unsure). Based on answers, output a ballpark hours estimate and a cost range. Include a user-adjustable hourly rate field defaulting to $200/hr. Briefly explain the methodology behind the estimate so the prospect understands it's directional. Professional palette, clean card layout, mobile-friendly, single index.html. Include a clear CTA at the end: "Ready to get started? Contact us." with a placeholder the user can replace with their actual contact or booking link. No frameworks.
USE CASE 04
Discovery Questionnaire
An intake form with branching logic that asks smart follow-up questions based on prior answers, then delivers a clean summary. Replaces a static Google Form with something that feels like the premium intake a high-end firm would have.
Build me a single-page HTML prospect discovery questionnaire for an accounting firm. The questionnaire should feel premium and consultative, not clinical. Start with the prospect's name, business name, email, and phone. Then ask about the business: entity type, industry, annual revenue range, number of employees, years in business. Then branch based on answers. If they have employees, ask about payroll provider and pain points with it. If entity type is S-corp or C-corp, ask about reasonable comp and distribution practices. If they mention multiple states, ask about nexus and compliance concerns. For everyone, ask: what's working in your current accounting setup, what's not working, what keeps you up at night financially, and what would a successful engagement with a new accountant look like in 12 months. At the end, show a clean summary of their answers that the prospect can review, and allow them to submit it to a provided email address (use a mailto link as a placeholder). Professional, warm, modern palette. Mobile-friendly. Single index.html, no frameworks.
USE CASE 05
Cash Flow Runway Visualizer
Cash on hand, monthly burn, expected receivables in — runway chart by month out. A tangible deliverable for every advisory conversation, especially useful for the clients who only know their cash position as "I think we're fine?"
Build me a single-page HTML cash flow runway visualizer for small business advisory clients. The user enters: current cash on hand, average monthly operating expenses (burn rate), projected monthly revenue for the next 12 months (with a simple input method — e.g., "same as current" with an adjustable percentage growth/decline, or an option to enter each month individually), and any known one-time cash events (receivables coming in, large expenses going out). The tool generates a 12-month cash position chart using Chart.js, with a clear "danger zone" line showing when cash drops below a user-adjustable minimum safe balance (default: 2 months of operating expenses). Color the monthly bars green (above danger zone), yellow (close), or red (below). Include a "what-if" revenue slider that lets the user adjust overall revenue by plus or minus 30% and watch the chart update. Display the runway in months as a prominent number at the top. Clean, modern palette suitable for client-facing advisory meetings. Mobile-friendly. Single index.html with Chart.js via CDN.
USE CASE 06
"Is Your Bookkeeping Healthy?" Diagnostic
A ten-question scored self-assessment producing a tailored recommendation. The interactive version of every "free download" PDF a firm puts on their site that prospects rarely finish — this one they actually complete.
Build me a single-page HTML interactive diagnostic quiz titled "Is Your Bookkeeping Healthy?" Ten yes/no questions, one at a time with a progress indicator, covering: bank reconciliations done monthly, books closed quarterly, A/R under 60 days, A/P current, payroll tax deposits on time, 1099s filed timely last year, financial statements available within a week of close, inventory accurate (or N/A), sales tax filed and paid on time (or N/A), profit margins reviewed monthly. Score each question yes=1, no=0, N/A=1. Based on total score out of 10, display one of four results: "Healthy" (9-10), "Mostly on track" (7-8), "Needs attention" (4-6), or "Cleanup recommended" (0-3). Each result gets a 2-3 sentence tailored recommendation. At the end, include an optional email capture field (use a simple input with a note that this needs to be wired to an actual form handler like Formspree or similar). Include a clear CTA button with placeholder text the user can customize. Professional, clean palette, mobile-friendly, single index.html, no frameworks.
USE CASE 07
Subscription Tier Calculator
Prospect picks services they need across three tiers, tool returns the right tier and monthly price. A replacement for the "let me send you a proposal" ritual — works as a real-time scoping tool on a prospect call or embedded on the pricing page of a firm's website.
Build me a single-page HTML subscription tier calculator for an accounting firm. The firm has three service tiers: Tier 1 (Foundation - bookkeeping, monthly financials, year-end taxes), Tier 2 (Growth - everything in Tier 1 plus quarterly advisory meetings, cash flow forecasting, tax planning), and Tier 3 (Scale - everything in Tier 2 plus monthly advisory meetings, fractional CFO services, strategic planning). Display all services across all tiers as draggable chips or checkboxes. The prospect selects everything they need. The tool determines which tier includes everything they selected and recommends it with a monthly price. Include user-editable service names and tier prices near the top so the accountant can customize the tool for their specific firm before publishing. If the prospect selects services that span multiple tiers, recommend the highest tier that covers all their needs. Display the recommendation with a clear explanation, full list of what's included, and a CTA button linking to a placeholder booking link. Clean, modern palette, mobile-friendly, single index.html, no frameworks.
