WHY NOT WIX OR SQUARESPACE?

To put your app at calc.yourpractice.com, you need to add a CNAME record to your domain's DNS. Wix, Squarespace, GoDaddy Website Builder, Weebly — these platforms own your DNS. They won't let you point a subdomain anywhere outside their walls. The fix is simple: keep your website where it is, but transfer your domain to a real registrar (GoDaddy registrar, Namecheap, Cloudflare). One-time setup. You'll thank yourself for the rest of your career.

One project folder, one prompt, one working tool. Then we iterate until it looks the way you want it to look. The loan calculator is a starting point — the workflow is what matters.

STAGE 2 · PART 2 · APRIL 28

Ship It Live

Tuesday's call. From your laptop to the public internet on your own domain.

GitHub hosts. Your DNS routes. HTTPS provisions automatically. Five steps and your app is live at calc.yourpractice.com (or whatever subdomain you choose).

1

Push to GitHub

Claude Code handles the entire git workflow in one prompt.

PROMPT Initialize this folder as a git repo, create a new GitHub repo called loan-calculator under my account, and push everything. Make it public for now. Give me the URL when done.
2

Turn on GitHub Pages

On the repo page → Settings → Pages. Source: Deploy from a branch. Branch: main / root. Save. Within 30 seconds you have a live URL at username.github.io/repo-name.

3

Add your custom subdomain

In GitHub: Settings → Pages → Custom domain → enter calc.yourpractice.com. Save. (Red X is expected — DNS hasn't been pointed yet.)

4

Add a CNAME at your registrar

Log into GoDaddy / Namecheap / Cloudflare. DNS settings → add a CNAME record:

CNAME RECORD Type: CNAME Name: calc (just the subdomain prefix) Value: username.github.io (no https://, no trailing slash) TTL: default
5

Enforce HTTPS and you're live

Wait a few minutes for DNS to propagate. Once GitHub shows the green checkmark, check "Enforce HTTPS." Free SSL via Let's Encrypt activates within a minute. Visit https://calc.yourpractice.com. There it is.
